# VERA Benchmark Problem 4：三维 HZP 3×3 组件控制棒价值（3D HZP 3×3 Assembly Control Rod Worth）— 明确化输入描述

> **来源**：CASL-U-2012-0131-004《VERA Core Physics Benchmark Problems》Problem #4  
> **反应堆**：WBN1 Cycle 1 初始堆芯中心 3×3 区域  
> **状态**：寿期初（BOL）/ 热态零功率（HZP）/ 565 K 等温条件  
> **组件内坐标**：17×17 栅格使用 **1-based `(row, col)`**，左上角为 `(1,1)`，中心仪表管为 `(9,9)`。  
> **组件阵列坐标**：3×3 组件阵列同样使用 **1-based `(assembly_row, assembly_col)`**，左上组件为 `(1,1)`，中心组件为 `(2,2)`。

> **本版建模约定**：本题不是把 9 个二维 pin map 沿轴向简单拉伸。下文将模型拆分为：①九个组件中连续存在的燃料棒、导向管和仪表管路径；②只在有限高度存在的 Pyrex、套管塞和移动 RCCA；③不切断棒/管路径的定位格架外框覆盖层；④取代整个详细棒栅格横截面的均匀化 nozzle、堆芯板和上下慢化剂缓冲层。原始表格未唯一规定的端塞细节必须标注为建模约定，不得伪装成精确基准事实。

> **给 Agent 的重要指令**：首先建立 **基础工况**（RCCA 毒物底面 `z=257.9 cm`）并完成几何校验；随后才批量生成 0–230 步的 11 个棒位工况。不同棒位只应平移中心组件的 RCCA 轴向剖面，不得重新生成或改变其余 8 个组件、材料、格架、Pyrex 或套管塞布置。

---

## 1. 目的（Purpose）

本基准题在 Problem 3 单组件三维模型基础上增加：

1. 九个燃料组件组成的 3×3 多组件区域；
2. 两种燃料富集度和三种组件类型；
3. 四个边组件中的 Pyrex 可燃毒物；
4. 角组件和边组件上部的套管塞；
5. 中心组件中可沿轴向移动的 24 束混合 AIC/B₄C RCCA；
6. 不同棒位下的 `k-effective`、微分棒价值（DRW）、积分棒价值（IRW）和功率分布。

本题不进行燃耗或热工水力反馈计算。所有组件处于统一 HZP 工况。径向外边界采用反射边界，轴向上下外边界采用真空边界。

### 2.2 组件中心坐标约定

若 3×3 区域几何中心设为 `(x,y)=(0,0)`，组件栅距为 `21.50 cm`，则可采用：

```text
x_center = (assembly_col - 2) × 21.50 cm
y_center = (2 - assembly_row) × 21.50 cm
```

因此：

- 左上组件 `(1,1)` 中心为 `(-21.50, +21.50) cm`；
- 中心组件 `(2,2)` 中心为 `(0,0) cm`；
- 右下组件 `(3,3)` 中心为 `(+21.50, -21.50) cm`。

完整 3×3 区域宽度为 `64.50 cm`，若以中心为原点，径向外边界位于 `x/y=±32.25 cm`。

> **关键边界语义**：反射边界只设置在整个 3×3 区域的最外侧。九个组件之间的内部界面必须是普通传输界面，不得给每个组件单独设置反射边界，否则组件间中子耦合会被错误切断。

### 2.3 组件间隙

单组件 pin lattice 的宽度为：

```text
17 × 1.26 = 21.42 cm
```

组件栅距为 `21.50 cm`，因此每个组件在每个径向外侧保留 `0.04 cm` 半间隙；两个相邻详细 lattice 边缘之间的总水隙为 `0.08 cm`。该间隙填本工况含硼水。

## 7. 三维路径语义

二维 pin map 只决定每个 `(assembly, row, col)` 位置属于哪一类轴向路径，不决定该路径在全高使用同一个 universe。

- **燃料棒路径 `F`**：依次经过下肩水区、下端塞、活性燃料、上端塞、氦气腔和上肩水区。
- **普通导向管路径 `G`**：在详细组件区内始终保留 Zircaloy-4 导向管壁，内部通常为含硼水。
- **Pyrex 路径 `P`**：只在 Pyrex 毒物段使用环形 Pyrex 截面；毒物上方使用 Pyrex 上部氦气腔截面；其余高度恢复普通水导向管。
- **套管塞路径 `T`**：只在 `383.31–394.31 cm` 内含 SS304 实心塞；其余高度是普通水导向管。
- **RCCA 路径 `RCCA`**：控制棒的 AIC、B₄C、上部氦气腔和端塞剖面随棒位整体平移；棒件不在该高度时恢复普通水导向管。
- **仪表管路径 `I`**：保持水填充的 Zircaloy-4 仪表管，不含 Pyrex、套管塞、RCCA 或 SS304 仪表套管。

插入件只替换导向管的内部流道，不能删除 `0.561–0.602 cm` 的 Zircaloy-4 导向管壁，也不能替换导向管外慢化剂。

## 11. 定位格架

每个组件有 8 个格架，因此整个 3×3 区域有：

```text
9 × 8 = 72 grids
18 end grids
54 middle grids
```

| 参数 | 端部格架 | 中间格架 |
|---|---:|---:|
| 每组件数量 | 2 | 6 |
| 材料 | Inconel-718 | Zircaloy-4 |
| 单个格架质量 | 1017 g | 875 g |
| 高度 | 3.866 cm | 3.810 cm |
| 中心 z (cm) | 13.884, 388.2 | 75.2, 127.4, 179.6, 231.8, 284.0, 336.2 |

### 11.1 质量守恒外框

每个格架的质量在该组件的 289 个 pin cells 间平均分配：

```text
A_cell = grid_mass / (material_density × 289 × grid_height)
```

采用对称方形外框时：

```text
inner_side = sqrt(pin_pitch² - A_cell)
frame_thickness = (pin_pitch - inner_side) / 2
```

由给定密度得到的检查值约为：

| 格架 | `A_cell` | 外框厚度 |
|---|---:|---:|
| 端部 Inconel | 0.111142 cm² | 0.02245 cm |
| 中间 Zircaloy | 0.121138 cm² | 0.02451 cm |

这些派生值必须由质量、密度、高度和 pin 数确定性计算，不应作为通用 renderer 中的硬编码常数。

### 11.2 叠加规则

- 格架材料只占每个 pin cell 最外侧原本为慢化剂的区域。
- 燃料、氦隙、包壳、导向管壁、仪表管壁、Pyrex、套管塞和 RCCA 均继续贯穿格架高度。
- 格架应在所有 lattice loading 或局部插入件物化之后叠加。
- 不得把整个格架层填成 Inconel/Zircaloy，也不得在 24 个导向管周围另加未给出的 spacer sleeve。
- 相邻组件的格架分别属于各自组件；组件间 `0.08 cm` 总水隙不得被格架外框越界占据。

| 下 nozzle | 0.000–6.053 | 27.9217 vol% SS304 + 72.0783 vol% 本工况冷却剂 |

nozzle 体积分数由：

```text
f_SS = 6250 g / (8.00 g/cc × 21.50² cm² × nozzle_height)
```

确定。

### 12.2 横截面语义

- 这些区域是覆盖整个 3×3 计算横截面的均匀化 slab；也可按九个相邻组件 footprint 分区，但材料必须一致且不能留缝。

```text
new_z = reference_z + withdrawal_steps × 1.5875 cm
```

不得只移动 B₄C、只移动毒物而保持包壳不动，或根据固定轴向层把控制棒“吸附”到最近网格边界。

### 15.5 与详细组件区的裁剪

RCCA 只在中心组件的导向管详细区内显式渲染：

```text
6.053 ≤ z < 397.510 cm
```

- 低于 rodlet 下端的导向管内部为含硼水。
- 高于已建模 rodlet 顶部且仍在详细区内的部分恢复为含硼水。
- 超过 `397.510 cm` 的 RCCA 几何不与均匀化上 nozzle/堆芯板重复叠加。

- **全插（0 步）**：AIC `17.031–118.631 cm`，B₄C `118.631–377.711 cm`；B₄C 顶面恰到燃料栈顶。

## 18. 推荐装配顺序

1. 建立统一 HZP 材料和两种 UO₂ 富集度。
2. 建立燃料段、端塞、气腔、全水 pin cell、普通导向管和仪表管等基础 universe。
3. 建立 Pyrex 毒物段、Pyrex 上部气腔和套管塞内部剖面。
4. 建立 RCCA 的 AIC、B₄C、上部气腔和可选端塞内部剖面。
5. 建立 `C`、`E`、`R` 三类基础 17×17 assembly lattice，不把有限高度插入件沿全高固化进 pin map。
6. 根据固定结构边界和当前 RCCA 动态边界构造轴向分段。
7. 在各段中先应用 fuel profile、Pyrex/套管塞/RCCA 等 lattice loading。
8. 在最终 materialized universe 外侧叠加格架质量守恒方框。
9. 将三个 3D reusable assembly universe 按 `C/E/R` 组成 3×3 core lattice。
10. 在 nozzle、堆芯板和上下缓冲层切换为整平面 slab。
11. 最后设置 3×3 外边界、轴向真空边界、初始源和 tallies。

推荐的局部组合顺序：

```text
base pin/guide paths
→ assembly-specific finite inserts
→ current RCCA axial loading
→ mass-conserving spacer-grid overlay
→ 3D assembly universe
→ 3×3 core lattice
→ homogenized slabs / outer boundaries
```

## 20. 常见错误与阻断条件

| 错误 | 后果 |
|---|---|
| 给每个组件设置径向反射边界 | 九个组件彼此中子学隔离，失去 3×3 耦合 |

## 22. Tally 与功率网格

### 22.1 几何层与 tally 层分离

轴向 tally 网格不等于几何轴向层。几何必须包含所有真实材料/构件边界；裂变率 tally 可使用约 49 个轴向区段，并在格架区使用 3.810/3.866 cm 的薄层分辨。

### 22.2 基础工况输出

基础工况至少输出：

1. `k-effective`；
2. 九个组件的归一化组件功率；
3. 2376 根燃料棒的径向 pin-power 分布或其对称展开结果；
4. 平均轴向功率分布；
5. 轴向偏移（若求解流程支持）。

功率只在 UO₂ 燃料区域计分。Pyrex、RCCA、套管塞和结构材料中的吸收或沉积不得计作燃料功率。

### 22.3 棒位扫描输出

对基础工况以及 0、23、46、69、92、115、138、161、184、207、230 步分别计算 `k-effective`。

定义从较插入状态 `i` 到下一较抽出状态 `i+1` 的微分棒价值：

```text
DRW_i = (1/k_{i+1} - 1/k_i) × 10^5 pcm
```

以 230 步全抽工况为未插棒基准，积分棒价值：

```text
IRW_i = (1/k_230 - 1/k_i) × 10^5 pcm
```

上述符号约定下，插入控制棒的价值通常为负；如另行报告正的吸收价值绝对值，必须明确写为 `-DRW` 或 `-IRW`，不能混用符号。

基础工况 `z_bottom=257.9 cm` 的积分棒价值也相对 230 步工况计算。

### 23.3 棒位批量生成级

- 12 个状态使用同一个静态 3×3 几何合同；
- 只有 RCCA 轴向位置随状态变化；
- 相同状态重复生成的 plan/XML semantic hash 一致；
- 棒位上移量与 `steps×1.5875 cm` 一致；
- 230 步状态的 RCCA 不进入活性燃料区；
- 0 步状态的 B₄C 顶面为 `377.711 cm`。

## 25. 截面库路径

截面库路径属于运行环境，不属于 benchmark 物理输入。OpenMC 可通过环境变量指定：

```bash
export OPENMC_CROSS_SECTIONS=/path/to/cross_sections.xml
```

模型或自然语言输入中不得硬编码某台机器的 `cross_sections.xml` 绝对路径。若本地核数据缺少 AIC、B₄C、结构合金或热散射所需核素，应作为环境/核数据 blocker 报告，而不是静默删除对应材料成分。
