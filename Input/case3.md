# 算例3：C5G7-3D 四分之一堆芯 OpenMC 模型

请根据下面的自然语言描述，生成一个能够完整定义 C5G7-3D 四分之一堆芯的 OpenMC 模型，并同时给出必要的几何一致性校验和低计算成本运行诊断方案。本输入文件只提供自然语言建模约束，不提供可直接照抄的 Python 脚本、YAML、JSON 或其它结构化中间表示。生成模型时应由智能体自行把这些描述解析为材料、曲面、栅格、单元、宇宙、几何、设置和计数校验。

本算例用于测试 openmc-agent 是否能够从自然语言约束中恢复一个较大规模三维反应堆基准模型。目标不是进行高统计量物理基准验证，而是验证自动建模流程能否生成完整几何、导出 OpenMC XML、通过几何检查，并用低计算成本本征值运行排查明显建模错误。

## 一、基准题背景和适用范围

本算例对应 OECD/NEA C5G7 MOX 基准题的三维四分之一堆芯形式。该基准题用于考察输运程序在不对燃料棒以上尺度进行空间均匀化的情况下处理 UO2 与 MOX 组件界面、17×17 棒阵列、径向反射层和三维轴向边界的能力。

本输入文件要求建立的是未插棒的三维四分之一堆芯版本。堆芯径向由 2×2 个燃料组件构成，外侧有一圈水反射层。每个燃料组件是 17×17 棒位阵列。四个燃料组件拼接后，活性燃料棒位阵列尺寸为 34×34。轴向方向包含燃料有效区和顶部水反射层，底部采用反射边界，上部外表面采用真空边界。

C5G7 原始基准题使用七群宏观截面和均匀化的棒内材料语义，其中燃料、包壳和间隙在圆柱内被作为等效混合材料处理，圆柱外是慢化剂水。蒙特卡罗输运的常规做法是连续能模拟，因此本算例不使用 C5G7 七群宏观截面，而是采用连续能核素材料卡：在保持 C5G7 几何与排布不变的前提下，把每个材料区域替换为连续能核素组成、密度、温度和热散射处理。由于 C5G7 原始基准只给出七群均匀化截面、并未提供连续能核素组成，本算例给出的连续能材料数据是本算例为蒙卡可运行性确定的核素组成定义，不是 C5G7 基准标准数据；低成本运行结果只能用于程序诊断，不能当作高统计量 C5G7 物理基准验证。

## 二、总体建模目标

请建立一个 OpenMC 三维模型，使它能够完整表示 C5G7-3D 四分之一堆芯。模型至少应满足以下要求。

第一，能够生成 34×34 的活性棒位阵列，并保证它由四个 17×17 燃料组件拼接而成。

第二，能够区分 UO2 组件和 MOX 组件，并按照四分之一堆芯中 2×2 的组件排布放置。

第三，能够在燃料组件外侧设置一圈水反射层。水反射层的厚度等于一个组件宽度，也就是 21.42 cm。

第四，能够设置燃料有效区和顶部水反射层两个轴向区域。燃料有效高度为 192.78 cm，顶部水反射层高度为 21.42 cm，因此总高度为 214.20 cm。

第五，能够把反射边界和真空边界清楚地映射到 OpenMC 的曲面边界条件。对称面应为反射边界，外部泄漏面应为真空边界。

第六，能够通过 OpenMC 的 XML 导出、几何检查和低计算成本本征值运行诊断，确认没有明显的 XML 解析错误、几何重叠、未闭合区域、lost particle 或 ERROR 级错误。

## 三、基本几何尺寸

所有长度单位均为 cm。

每个燃料棒栅元的边长为 1.26 cm。燃料棒、导向管和裂变室的圆柱半径均取 0.54 cm。每个燃料组件包含 17×17 个棒位，因此组件边长为 21.42 cm。四分之一堆芯中的活性燃料组件区由 2×2 个组件构成，所以活性组件区边长为 42.84 cm。外侧水反射层厚度为一个组件宽度，即 21.42 cm。把 2×2 活性组件区和外侧一圈水反射层一起纳入堆芯级径向方形区域时，该四分之一堆芯径向边长为 64.26 cm。

轴向方向从底部到顶部分为两个连续区间。底部到 192.78 cm 是燃料有效区，在该区间内保留径向燃料组件和径向水反射层。192.78 cm 到 214.20 cm 是顶部水反射层，在该区间内整个横截面都应填充为水。顶部水反射层上方是真空边界。燃料有效区下方是反射边界。

## 四、径向组件排布

四分之一堆芯的活性燃料组件区包含四个 17×17 组件，排布为两个 UO2 组件和两个 MOX 组件。按工程图视角描述时，约定“上方”为北，“左侧”为西。活性 2×2 组件区的左上角是 UO2 组件，右上角是 MOX 组件，左下角是 MOX 组件，右下角是 UO2 组件。

在活性 2×2 组件区的外侧设置一圈水反射层。用 3×3 的组件宽度区域描述整个四分之一堆芯时，左上 2×2 区域为燃料组件，右侧一列为水反射层，下侧一行为水反射层，右下角也是水反射层。这样得到的径向整体区域宽度为三个组件宽度，即 64.26 cm。

OpenMC 中采用非负坐标或中心坐标。无论使用哪种坐标约定，都必须保证同一物理关系：两个相交的内侧对称面为反射边界，外侧水反射层的外表面为真空边界。采用非负坐标时，左侧和上侧作为对称面，右侧和下侧作为外侧真空边界；采用以堆芯角点或中心为基准的其他坐标时，也必须在模型说明中明确每个边界面的物理含义，避免把对称面和外表面弄反。

## 五、棒位类型和材料区域

模型中至少应包含七类棒位或区域：UO2 燃料棒、4.3% MOX 燃料棒、7.0% MOX 燃料棒、8.7% MOX 燃料棒、导向管、裂变室以及水反射层。

几何上沿用 C5G7 简化：每个燃料棒位分为两部分，半径 0.54 cm 以内是燃料材料，半径 0.54 cm 以外到 1.26 cm 方形栅元边界之间是水。与原始七群基准不同，这里的燃料、导向管和裂变室圆柱内不再使用预均匀化的等效混合截面，而是改用连续能核素材料卡，分别填入 UO2、MOX、导向管材料和裂变室材料的核素组成。导向管和裂变室也使用半径 0.54 cm 的圆柱区域，圆柱外仍为水。纯水反射层区域全部为水。

本算例采用连续能模式。OpenMC 材料应使用 `set_density("g/cm3", ...)`、`add_nuclide(...)` 以及在存在热散射时使用 `add_s_alpha_beta(...)` 定义连续能核素组成，不要使用 `add_macroscopic`，也不要设置 `density_unit="macro"`。OpenMC 设置无需显式指定 `energy_mode`（连续能为默认值）。运行时通过环境变量 `OPENMC_CROSS_SECTIONS` 指向本机连续能库 `/home/wbd/openmc_data/endfb-vii.1-hdf5/cross_sections.xml`（ENDF/B-VII.1 HDF5），确保 U-235、U-238、Pu-238、Pu-239、Pu-240、Pu-241、Pu-242、O-16、Zr、Sn、H-1 以及水的热散射核 `c_H_in_H2O` 均可用。

各材料的连续能核素组成如下。温度统一取 C5G7 冷态 293.6 K。组成基均为原子百分数（ao），密度单位为 g/cm3。这些数值是本算例为 C5G7 几何确定的连续能核素组成定义，不是 C5G7 七群基准的标准定义；建模时直接使用。

UO2 燃料（材料名 `uo2`）：U-235 富集度 3.3 wt%（在铀中），U-238 为铀的余量，O-16 与铀按 UO2 化学计量配平（O/U 原子比为 2.0）；密度 10.0；温度 293.6 K；无热散射。

4.3% MOX 燃料（材料名 `mox43`）：钚含量 4.3 wt%（占重金属），钚同位素组成按在钀中 Pu-238 1.9 ao%、Pu-239 58.0 ao%、Pu-240 24.0 ao%、Pu-241 11.2 ao%、Pu-242 4.9 ao%，其余为贫铀（U-238 为主、U-235 约 0.25 ao%），O-16 按 (U+Pu)O2 配平；密度 10.0；温度 293.6 K。

7.0% MOX 燃料（材料名 `mox7`）：钚含量 7.0 wt%（占重金属），钚同位素组成同 `mox43`，其余贫铀，O-16 按 (U+Pu)O2 配平；密度 10.0；温度 293.6 K。

8.7% MOX 燃料（材料名 `mox87`）：钚含量 8.7 wt%（占重金属），钚同位素组成同 `mox43`，其余贫铀，O-16 按 (U+Pu)O2 配平；密度 10.0；温度 293.6 K。

导向管（材料名 `guide_tube`）：取 Zircaloy-4，采用 Zr 98.23 wo%、Sn 1.45 wo%、Fe 0.21 wo%、Cr 0.10 wo%；密度 6.56；温度 293.6 K。

裂变室（材料名 `fiss_chamber`）：C5G7 七群基准中裂变室为特殊等效材料，连续能版本无标准定义；本算例将裂变室取为与导向管相同的 Zircaloy-4，建模时直接复用 `guide_tube` 材料定义；密度 6.56；温度 293.6 K。

水（材料名 `water`）：H-1 与 O-16 按 H2O 化学计量配平（H-1 66.67 ao%、O-16 33.33 ao%）；密度 0.997；温度 293.6 K；热散射 `c_H_in_H2O`。水慢化剂、顶部水反射层和径向水反射层均使用该材料。

上述连续能核素组成不是 C5G7 七群基准的标准数据：C5G7 基准只给出七群均匀化截面，并未给出连续能核素组成、密度或温度。所列 UO2 富集度、MOX 钚含量与钚同位素组成、Zircaloy-4 合金组成、水密度和热散射核名均为本算例确定的连续能核素组成取值，用于完整定义 OpenMC 连续能模型；由于这些取值并非 C5G7 七群基准数据，所得 keff 不能与 C5G7 七群参考 keff 直接比较。本算例是未插棒构型，不使用控制棒材料。

## 八、四分之一堆芯棒位总数

由于四分之一堆芯中有两个 UO2 组件和两个 MOX 组件，活性燃料组件区的总棒位为 1156 个。燃料棒总数为 1056 根，其中 UO2 燃料棒 528 根，4.3% MOX 燃料棒 128 根，7.0% MOX 燃料棒 200 根，8.7% MOX 燃料棒 200 根。非燃料棒位包括 96 个导向管位置和 4 个裂变室位置。

这些数量必须作为自动生成后的硬性校验项。若生成结果与这些数量不一致，应视为模型生成失败，而不是通过改变材料命名或忽略部分棒位来修正。

## 九、轴向分层和三维闭合

模型必须在轴向上闭合。燃料有效区从底部开始，延伸 192.78 cm。在该区间内，径向 2×2 燃料组件和外侧水反射层同时存在。顶部水反射层紧接燃料有效区，厚度 21.42 cm，从 192.78 cm 延伸到 214.20 cm。在顶部水反射层内，不应继续放置燃料棒、导向管或裂变室，整个横截面应为水。

底部边界为反射边界，顶部边界为真空边界。燃料区与顶部水反射层之间的内部轴向平面不应设置为真空或反射边界，它只是普通内部分界面。生成几何时应避免把顶部水反射层漏掉，也应避免燃料棒圆柱无限延伸到顶部水反射层内。

## 十、边界条件

该四分之一堆芯模型有两个径向对称面和两个径向外表面。两个对称面应采用反射边界，两个外表面应采用真空边界。径向外表面位于水反射层外侧，而不是活性燃料组件区外侧。底面采用反射边界，顶面采用真空边界。

在 OpenMC 生成过程中，应清楚记录每一个边界面的名称、方向、位置和边界类型。若采用矩形棱柱或多个平面组合定义整体区域，应确认反射平面和真空平面没有重复定义或互相覆盖。若使用水反射层单元覆盖外侧区域，应保证水反射层的外表面正好与真空边界相接。

## 十一、OpenMC 建模组织要求

openmc-agent 采用适合的 OpenMC Python API 组织方式，但生成结果必须满足以下语义。

每类棒位用独立的 pin universe 或等效组织方式表示。UO2、4.3% MOX、7.0% MOX、8.7% MOX、导向管和裂变室都必须能被单独识别和计数。水反射层用独立的全水区域或全水 universe 表示。

每个 17×17 组件应由对应的棒位描述生成。两个 UO2 组件应复用同一种 UO2 组件定义，两个 MOX 组件应复用同一种 MOX 组件定义。堆芯级组织应能表达 2×2 活性组件区和一圈水反射层。

使用 OpenMC RectLattice 时，应特别注意 OpenMC 的 lattice 行列顺序与工程图从上到下、从左到右描述之间的差异。生成器必须显式处理 y 方向顺序，不能把组件上下翻转后仍认为模型正确。不用 RectLattice 时，也必须保证每个棒位的实际空间位置与本描述一致。

模型应能够生成 OpenMC materials、geometry、settings 和 tallies，并导出 XML 文件。tallies 为用于诊断的简单通量或裂变率 tallies，不要求完成高统计量 pin power 基准验证。

## 十二、几何一致性校验清单

生成模型后，必须进行以下校验。

首先，检查每个 UO2 组件和 MOX 组件都是 17×17 棒位，四个组件拼接后的活性组件区是 34×34 棒位。

其次，检查棒位栅距为 1.26 cm，圆柱半径为 0.54 cm，圆柱半径小于半个栅距，圆柱不会超出方形栅元。

第三，检查两个 UO2 组件和两个 MOX 组件的排布没有旋转、镜像或上下颠倒错误。特别要检查裂变室和导向管位置是否位于预期行列。

第四，检查堆芯级总数是否为 UO2 燃料棒 528 根、4.3% MOX 燃料棒 128 根、7.0% MOX 燃料棒 200 根、8.7% MOX 燃料棒 200 根、导向管 96 个、裂变室 4 个、燃料棒总数 1056 根。

第五，检查活性燃料组件区边长为 42.84 cm，整个包含水反射层的四分之一堆芯径向边长为 64.26 cm。

第七，检查反射边界只位于四分之一堆芯的对称面和底面，真空边界只位于径向外表面和顶面。

## 十三、常见歧义和修复要求

生成过程中出现 lattice 或 universe 填充歧义时，应优先通过显式坐标、显式边界和显式分区修复，而不是依赖 OpenMC 默认外延。特别要处理以下问题。

OpenMC 的 RectLattice 行顺序与工程图行顺序不同，必须在生成时说明并修正索引映射。

堆芯级 3×3 区域中的水反射层作为全水组件宽度块处理，或用显式水 cell 处理；无论哪种方法，都必须保证它覆盖活性组件外侧的一整圈水反射区域。

燃料区和顶部水反射层使用不同的轴向 cell 或 universe 区分。不能让 pin universe 中的圆柱区域无界延伸到顶部水反射层。

反射边界和真空边界的平面应只定义在整体模型外表面。内部组件边界、棒位边界和燃料区与顶部反射层之间的分界面不应带有真空或反射边界条件。

由于不同组件共享同一 pin surface 导致空间区域定义冲突时，应改为在局部 universe 内定义相对坐标 pin surface，或确保每个局部 universe 的曲面作用域符合 OpenMC 几何语义。
