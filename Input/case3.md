# 算例3：C5G7-3D 四分之一堆芯 OpenMC 模型

请根据以下自然语言描述，生成用于 OpenMC 模型自动生成与校验的结构化建模信息。要求保留源资料中的模型规模、边界关系、默认近似和校验边界，不要把低计算成本运行结果解释为物理量验证。

## 建模目标

建立 C5G7-3D 四分之一堆芯 OpenMC 模型，用于验证智能体流程能否支持较大规模模型生成、堆芯级棒阵列组织、轴向分层结构、组件排布、边界条件设置、OpenMC lattice/universe 层级和几何一致性校验。

该算例主要检验：

- 34×34 活性棒阵列是否能够被正确生成；
- 2×2 个 17×17 组件及水反射层是否能够被组织为四分之一堆芯模型；
- 轴向燃料区和顶部水反射层是否闭合；
- 反射/真空边界关系是否能够被明确记录为 OpenMC surface boundary type；
- 自动生成的大规模 OpenMC Python 脚本是否能够通过静态检查、XML 导出检查、几何检查和低计算成本 eigenvalue 运行诊断。

## 已知模型信息

- 模型名称：C5G7-3D 四分之一堆芯模型。
- 径向区域：
  - 2×2 个 17×17 组件；
  - 水反射层。
- 轴向区域：
  - 燃料区；
  - 顶部水反射层。
- 活性棒阵列：34×34。
- 栅元边长：1.26 cm。
- 圆柱半径：0.54 cm。
- 燃料有效高度：192.78 cm。
- 顶部水反射层高度：21.42 cm。
- 源资料记录的旧格式输入规模：4848 行；该数字仅作为模型复杂度参考，不应作为 OpenMC 脚本行数要求。

## 棒位统计

模型包含以下棒位和材料类型：

- 528 根二氧化铀（UO₂）燃料棒；
- 128 根 4.3% 混合氧化物（MOX）燃料棒；
- 200 根 7.0% MOX 燃料棒；
- 200 根 8.7% MOX 燃料棒；
- 96 个导向管位置；
- 4 个裂变室位置；
- 燃料棒总数：1056 根。

## OpenMC 几何组织要求

请在生成模型时考虑以下组织方式：

- 以 34×34 活性棒阵列为核心建立堆芯径向结构。
- 使用 pin universe、17×17 组件 `openmc.RectLattice`、2×2 堆芯级 lattice 或等价分层 universe 组织结构。
- 组织 2×2 个 17×17 组件，并在外侧设置水反射层。
- 使用 z 平面定义燃料区和顶部水反射层的轴向高度，并确认轴向区间闭合。
- 棒位类型应至少包含：
  - UO₂ 燃料棒；
  - 4.3% MOX 燃料棒；
  - 7.0% MOX 燃料棒；
  - 8.7% MOX 燃料棒；
  - 导向管；
  - 裂变室；
  - 水反射层。
- 保证棒位数量统计与源资料一致。
- 使用 `openmc.Geometry`、`openmc.Settings`、`openmc.Tallies` 和 `openmc.Model(...).export_to_xml()` 形成可导出的模型脚本。

## 运行诊断要求

OpenMC 低计算成本诊断应满足：

- 使用 `settings.run_mode = "eigenvalue"`。
- 使用少量 batches、inactive batches 和 particles，例如 15 个 batches 量级；该设置仅用于模型接受性检查。
- 使用位于有效裂变区内的 `openmc.IndependentSource`，并检查源空间不落入水反射层或几何外部。
- OpenMC 日志中不应出现 XML 解析错误、几何重叠、lost particle 或 ERROR 级错误。
